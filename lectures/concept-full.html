1. Introduction to Graphs

Definition 1.1: A graph G = (V, E) consists of:
  • V: a finite set of vertices (or nodes)
  • E: a set of edges connecting pairs of vertices

Notation & Terminology

  • Order: |V| = number of vertices
  • Size: |E| = number of edges
  • Adjacent vertices: Connected by an edge
  • Incident edge: Edge connecting to a vertex
  • Neighbors: Set of all adjacent vertices

Example 1.1:

Consider graph G with:

  • V = {A, B, C, D}
  • E = {AB, AC, BC, CD}

Analysis:

  • Order = 4 vertices
  • Size = 4 edges
  • A is adjacent to B and C
  • Neighbors of C: {A, B, D}

2. Degree of a Vertex

Definition 2.1: The degree of a vertex v, denoted deg(v), is the number of edges incident to v.
Theorem 2.1 (Handshaking Lemma):
For any graph G = (V, E): Σv∈V deg(v) = 2|E|

Proof: Each edge contributes 2 to the total degree sum (one for each endpoint). Therefore, summing all degrees counts each edge exactly twice.
Corollary 2.1: In any graph, the number of vertices with odd degree is even.

Proof: Let O = set of odd-degree vertices, E = set of even-degree vertices. By Handshaking Lemma: Σ deg(v) = 2|E| (even number)
Since Σv∈E deg(v) is even (sum of even numbers),
Σv∈O deg(v) must also be even.
An even sum of odd numbers requires an even count. ∎

Example 2.1:

Graph with degree sequence [3, 3, 2, 2, 2]

  • Sum of degrees: 3+3+2+2+2 = 12
  • Number of edges: 12/2 = 6
  • Odd degree vertices: 2 (even count ✓)

3. Types of Graphs

3.1 Directed vs Undirected

  • Undirected: Edges have no orientation (e.g., friendship)
  • Directed (Digraph): Edges have direction (e.g., Twitter follows)
  • In digraphs: in-degree and out-degree

3.2 Weighted Graphs

  • Each edge e has a weight w(e) ∈ ℝ
  • Represents cost, distance, capacity, etc.
  • Used in shortest path problems
Definition 3.1: A simple graph has:
  • No multiple edges between same vertices
  • No self-loops (edges from vertex to itself)
Definition 3.2: A complete graph Kn is a simple graph where every pair of distinct vertices is connected by an edge.
Number of edges in Kn: |E| = n(n-1)/2 = C(n,2)

Example 3.1:

Complete graphs:

  • K3: Triangle (3 vertices, 3 edges)
  • K4: 4 vertices, 6 edges
  • K5: 5 vertices, 10 edges
  • Kn: n vertices, n(n-1)/2 edges

4. Special Graph Types

Definition 4.1: A bipartite graph G = (V, E) has vertex set V = A ∪ B where:
  • A ∩ B = ∅
  • Every edge connects a vertex in A to a vertex in B
Theorem 4.1: A graph is bipartite if and only if it contains no odd-length cycles.
Definition 4.2: A tree is a connected graph with no cycles.
Theorem 4.2: For a tree T with n vertices: |E| = n - 1

Example 4.1:

Real-world applications:

  • Bipartite: Job assignments (workers ↔ jobs)
  • Tree: Organization hierarchy, file systems
  • Complete: Round-robin tournaments
  • DAG: Task dependencies, course prerequisites

5. Paths and Connectivity

Definition 5.1: A path from vertex u to vertex v is a sequence of vertices (v0, v1, ..., vk) where:
  • v0 = u and vk = v
  • Each consecutive pair is connected by an edge
  • All vertices are distinct (except possibly u = v)
Definition 5.2: A cycle is a path where the first and last vertices are the same, with length ≥ 3.
Definition 5.3: A graph is connected if there exists a path between every pair of vertices.

Distance and Diameter

  • Distance d(u,v): Length of shortest path from u to v
  • Diameter: maxu,v∈V d(u,v)
  • Radius: minv∈V maxu∈V d(u,v)
Theorem 5.1: A graph with n vertices and more than n-1 edges must contain at least one cycle.

Practice Problems

Problem 1:

A graph has 8 vertices with degrees: 3, 3, 4, 4, 5, 5, 6, x. Find x.

Solution

By Handshaking Lemma: sum of degrees = 2|E| (must be even)

Current sum: 3+3+4+4+5+5+6 = 30

30 + x must be even, so x must be even

Also, x ≤ 7 (max degree in simple graph with 8 vertices)

Answer: x = 2, 4, or 6

Problem 2:

Prove: In any group of people, there are at least two people who know the same number of others.

Solution

Model as graph: vertices = people, edges = acquaintances

With n people, degrees range from 0 to n-1

If someone knows 0 people, no one can know all n-1 others

So only n-1 possible degree values for n people

By Pigeonhole Principle: at least two people have same degree ∎

Problem 3:

Can a simple graph have degree sequence [4, 4, 3, 3, 2]?

Solution

Sum of degrees: 4+4+3+3+2 = 16 (even ✓)

Number of edges: 16/2 = 8

5 vertices, max edges in simple graph: C(5,2) = 10 ✓

Check degree constraint: Each vertex degree ≤ 4 (n-1) ✓

Answer: Yes, it's possible

Problem 4:

A connected graph has 10 vertices and 12 edges. What's the minimum number of cycles?

Solution

A tree with 10 vertices has exactly 9 edges (no cycles)

Each additional edge creates at least one cycle

12 - 9 = 3 extra edges

Answer: At least 3 cycles

Problem 5:

Determine if K5 (complete graph on 5 vertices) is bipartite.

Solution

K5 contains triangles (3-cycles)

By Theorem 4.1: bipartite ⟺ no odd cycles

Since K5 has 3-cycles, it's not bipartite

Answer: No, K5 is not bipartite

Problem 6:

Find the diameter of a complete graph Kn (n ≥ 2).

Solution

In Kn, every pair of vertices is directly connected

Distance between any two vertices = 1

Diameter = maximum distance = 1

Answer: diameter(Kn) = 1

Chapter Summary

  1. Graphs model pairwise relationships: G = (V, E)
  2. Degree is fundamental: Handshaking Lemma relates degrees to edges
  3. Odd degree vertices always come in even numbers
  4. Special types: complete, bipartite, trees each have unique properties
  5. Trees are minimal connected graphs: |E| = n - 1
  6. Connectivity and paths form basis for many algorithms
