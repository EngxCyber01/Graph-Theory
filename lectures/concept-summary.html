Module 01: Quick Review

Essential Concepts at a Glance

← Return to Modules

Core Definitions

  • Graph: G = (V, E) - set of vertices connected by edges
  • Order: |V| = number of vertices
  • Size: |E| = number of edges
  • Degree: Number of edges incident to a vertex

Graph Types

Type Description Example Use
Undirected Edges have no direction Friendships, roads
Directed Edges have specific direction Web links, follows
Weighted Edges have numerical values Distances, costs
Complete (Kn) All vertices connected Tournaments
Tree Connected, acyclic Hierarchies
DAG Directed, no cycles Task scheduling

Key Formulas

Handshaking Lemma: Σ deg(v) = 2|E|
Complete Graph Kn: |E| = n(n-1)/2
Tree with n vertices: |E| = n - 1

Essential Terminology

  • Path: Sequence of vertices with no repetition
  • Cycle: Path that starts and ends at same vertex
  • Connected: Path exists between every pair of vertices
  • Component: Maximal connected subgraph
  • Distance: Length of shortest path between vertices
  • Diameter: Maximum distance in the graph

Quick Facts

  • Number of vertices with odd degree is always even
  • Trees have exactly n-1 edges for n vertices
  • Complete graph K5 has 10 edges
  • Adding edge to tree creates exactly one cycle

Quick Practice

  1. Graph with 5 vertices, max edges? Answer: 10
  2. Degrees [2,3,3,4], how many edges? Answer: 6
  3. Can odd degree count be 5? No (must be even)
  4. Facebook uses which type? Undirected
  5. Twitter uses which type? Directed

Key Takeaways

  1. Graphs model relationships: vertices and edges
  2. Choose type based on application needs
  3. Degree is fundamental for analysis
  4. Handshaking lemma is a powerful counting tool
  5. Trees are special: connected and acyclic
