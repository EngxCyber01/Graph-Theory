Module 04: Quick Review

Spanning Trees at a Glance

← Return to Modules

Core Definition

Spanning Tree: A subgraph that:

  • Is a tree (connected, acyclic)
  • Includes all vertices
  • Has exactly n-1 edges for n vertices

Minimum Spanning Tree (MST)

MST: Spanning tree with minimum total edge weight

Applications: Network design, clustering, approximation algorithms

Two Classic Algorithms

Kruskal's Algorithm

  1. Sort edges by weight
  2. Add cheapest edge that doesn't create cycle
  3. Repeat until n-1 edges

Time: O(m log m)

Best for: Sparse graphs

Prim's Algorithm

  1. Start from any vertex
  2. Add cheapest edge to tree
  3. Repeat until all vertices included

Time: O(m log n) with heap

Best for: Dense graphs

Key Properties

Cut Property: Minimum weight edge crossing a cut is in some MST
Cycle Property: Maximum weight edge in a cycle is not in any MST
Uniqueness: If all edge weights distinct, MST is unique

Quick Example

Graph with edges and weights:

  • AB: 4, AC: 3, BC: 2, BD: 5, CD: 1

Kruskal's order: CD(1) → BC(2) → AC(3) → Stop (4 vertices, 3 edges)

MST weight: 1 + 2 + 3 = 6

Quick Facts

  • Every connected graph has a spanning tree
  • Number of spanning trees grows exponentially
  • Complete graph Kn has nn-2 spanning trees (Cayley's formula)
  • Both Kruskal and Prim are greedy algorithms
  • MST may not be unique if edge weights repeat
MST weight ≤ any other spanning tree weight

Key Takeaways

  1. Spanning trees connect all vertices with minimum edges
  2. MST minimizes total edge weight
  3. Kruskal: sort edges, avoid cycles
  4. Prim: grow tree from single vertex
  5. Both algorithms guaranteed to find optimal MST
