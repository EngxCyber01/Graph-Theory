Essential Concepts at a Glance

Core Definitions

  • Permutation: Ordered arrangement - P(n,r) = n!/(n-r)!
  • Combination: Unordered selection - C(n,r) = n!/(r!(n-r)!)
  • Factorial: n! = n×(n-1)×(n-2)×...×2×1
  • Relationship: P(n,r) = C(n,r) × r!

Quick Decision Guide

Question Use This Formula
Does order matter? Permutation P(n,r) = n!/(n-r)!
Does order NOT matter? Combination C(n,r) = n!/(r!(n-r)!)
Can items repeat? With repetition nr (perm) or C(n+r-1,r) (comb)
Arranging in circle? Circular perm (n-1)!

Key Formulas Reference

Permutations

  • All n objects: n!
  • r from n: P(n,r) = n!/(n-r)!
  • With repetition: nr
  • Circular: (n-1)!
  • With identical items: n!/(n₁!×n₂!×...×nₖ!)

Combinations

  • r from n: C(n,r) = n!/(r!(n-r)!)
  • With repetition: C(n+r-1, r)
  • Symmetry: C(n,r) = C(n,n-r)
  • Pascal's Identity: C(n,r) = C(n-1,r-1) + C(n-1,r)

Quick Examples

Example 1: Basic Permutation

Q: 5 people in a line?

A: 5! = 120 ways

Example 2: Partial Permutation

Q: Top 3 from 8 runners?

A: P(8,3) = 8×7×6 = 336

Example 3: Basic Combination

Q: Choose 4 from 10?

A: C(10,4) = 210

Example 4: Committee Selection

Q: 3 from 7 men, 2 from 5 women?

A: C(7,3) × C(5,2) = 35 × 10 = 350

Common Patterns

Problem Type Approach Example
Adjacent items Treat as unit, multiply by internal (n-k+1)! × k!
Non-adjacent items Arrange others, place in gaps Arrange × P(gaps,items)
At least k Use complement: Total - Less than k C(n,r) - ΣC(n,i) for i<k
Distribute items Stars and bars C(n+k-1, k-1)

Counting Principles

  • Multiplication Rule: Independent events → multiply
  • Addition Rule: Mutually exclusive events → add
  • Complementary Counting: Total - Unwanted cases
  • Inclusion-Exclusion: |A∪B| = |A| + |B| - |A∩B|

Problem-Solving Strategy

  1. Identify: Does order matter? Repetition allowed?
  2. Count: Total number of items and selections
  3. Check: Any restrictions or conditions?
  4. Apply: Choose appropriate formula
  5. Multiply: Handle independent stages
  6. Verify: Does answer make intuitive sense?

Quick Tips

  • Arrangements/Rankings → Permutation
  • Committees/Groups → Combination
  • If calculating C(n,k) is hard, try C(n,n-k) instead
  • Break complex problems into stages
  • Complement method often simpler for "at least" problems
  • Always check: n ≥ r for both P(n,r) and C(n,r)
