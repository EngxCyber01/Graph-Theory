Three Main Representations

Method Space Edge Check Best For
Adjacency Matrix O(n²) O(1) Dense graphs
Adjacency List O(n+m) O(deg(v)) Sparse graphs
Edge List O(m) O(m) Simple storage

Adjacency Matrix

n × n matrix A where A[i][j] = 1 if edge (i,j) exists, 0 otherwise

  • Pros: Fast edge lookup O(1), simple operations
  • Cons: O(n²) space even for sparse graphs
  • Properties: Symmetric for undirected graphs

Adjacency List

Array of lists: each vertex stores list of neighbors

  • Pros: Space-efficient O(n+m), fast iteration
  • Cons: Slower edge check O(deg(v))
  • Best for: Most real-world graphs (sparse)

Quick Comparison

Dense Graph: m ≈ n² → Use Matrix
Sparse Graph: m ≪ n² → Use List

Quick Decision Guide

  • Need fast edge checks? → Adjacency Matrix
  • Save memory? → Adjacency List
  • Iterate neighbors often? → Adjacency List
  • Graph is dense? → Matrix
  • Graph is sparse? → List
Key Insight: Social networks, web graphs, and most real-world graphs are sparse (m = O(n)), making adjacency lists the practical choice.

Quick Facts

  1. Matrix: diagonal elements represent self-loops
  2. Matrix power Ak: counts k-length paths
  3. List: sum of all list lengths = 2m (undirected)
  4. Weighted graphs: store weights instead of 1/0
  5. Directed graphs: lists store outgoing edges
